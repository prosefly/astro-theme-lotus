# Markdown

Lotus uses Astro content collections and MDX, so standard Markdown remains the
default authoring format for most docs pages.

## Headings

Use headings to create page structure and table-of-contents anchors.

```md
## Section title
### Subsection title
```

## Lists

Ordered and unordered lists render with the shared prose styles.

```md
- Bullet item
- Another bullet

1. First step
2. Second step
```

## Tables

Tables work well for compact API and token references.

```md
| Field | Purpose |
| --- | --- |
| `title` | Page heading |
```

## Code

Use fenced code blocks for commands, configuration, and examples.

```md
```sh
npm run build
```
```

## Blockquotes

Use blockquotes for quoted text or simple callouts when you do not need a
custom MDX component.

```md
> Keep examples short and accurate.
```

## Links

Use standard Markdown links for internal and external references.

```md
[Configuration](/docs/guide/configuration/)
[Astro](https://astro.build)
```
