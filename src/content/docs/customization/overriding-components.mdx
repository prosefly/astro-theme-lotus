Lotus is designed to provide a strong default shell while keeping project
customization possible. Prefer configuration and CSS tokens first; reach for
component overrides only when the structure or behavior needs to change.

## Start with configuration

Many common changes belong in `src/theme.config.ts`: nav labels, footer links,
sidebar structure, actions, social icons, fonts, radius, and theme defaults.

## Wrap shared components

For content components, create local wrappers that import from
`@prosefly/astro-components` and set project-specific defaults.

```astro
---
import { Callout } from '@prosefly/astro-components';
---

<Callout type="note">
  <slot />
</Callout>
```

## Replace layout pieces carefully

Lotus exposes a small, stable override surface through the `components`
configuration. Start with the smallest override that changes the behavior you
need.

```ts
lotus({
  components: {
    HeaderNavbar: './src/components/HeaderNavbar.astro',
    HeaderSocialIcons: './src/components/HeaderSocialIcons.astro',
    PageActions: './src/components/PageActions.astro',
    PageHeader: './src/components/PageHeader.astro',
    PageNavigation: './src/components/PageNavigation.astro',
    SearchDialog: './src/components/SearchDialog.astro',
    SiteBrand: './src/components/SiteBrand.astro',
    ThemeSwitch: './src/components/ThemeSwitch.astro',
  },
})
```

Override paths are resolved from the project root. The first supported override
points are:

| Component | Purpose |
|---|---|
| `SiteBrand` | Logo and product name in the header |
| `HeaderNavbar` | Main navbar links and navbar action buttons |
| `HeaderSocialIcons` | Social icon links in desktop and mobile header chrome |
| `SearchDialog` | Search trigger and dialog content as one unit |
| `ThemeSwitch` | Theme mode control in the header |
| `PageHeader` | Documentation page title, description, section label, and page actions placement |
| `PageActions` | Copy page, Markdown, and assistant actions |
| `PageNavigation` | Previous and next page links |

## Use theme components

Composable theme components are exported from the theme package. Import them
when you want to wrap Lotus behavior instead of replacing it from scratch.

```astro
---
import { SearchDialog, ThemeModeButton } from '@prosefly/astro-theme-lotus/components';
---

<div class="rounded-lg border border-(--lotus-border-muted) p-1">
  <SearchDialog />
</div>

<ThemeModeButton />
```

Search can also be composed from its trigger and content:

```astro
---
import {
  SearchDialogContent,
  SearchDialogTrigger,
} from '@prosefly/astro-theme-lotus/components';
---

<div class="rounded-lg border border-(--lotus-border-muted) p-1">
  <SearchDialogTrigger />
</div>
<SearchDialogContent />
```

The theme itself imports override points through internal virtual modules. The
public `components` export points to theme building blocks, so using it inside
an override will not recursively import the override file.

For example, a project can wrap the default page actions:

```astro
---
import { PageActions } from '@prosefly/astro-theme-lotus/components';
---

<div class="rounded-(--lotus-radius-md) bg-(--lotus-surface) p-1">
  <PageActions {...Astro.props} />
</div>
```

Available theme mode controls:

```ts
import {
  ThemeModeButton,
  ThemeModeSegmentedControl,
  ThemeModeSelect,
  ThemeModeSwitch,
} from '@prosefly/astro-theme-lotus/components';
```
