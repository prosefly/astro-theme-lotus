# Configuration

Set branding, navigation, actions, appearance, and footer content in
`src/theme.config.ts`.

## Example

```ts
import type { LotusThemeConfig } from '@prosefly/astro-theme-lotus';

const themeConfig = {
  site: { title: 'Astro Theme Lotus', description: 'A documentation theme for Astro.', logo: '/logo.svg' },
  appearance: { accent: 'indigo', gray: 'zinc', fontSans: 'Inter, ui-sans-serif, system-ui, sans-serif', fontMono: 'JetBrains Mono, ui-monospace, SFMono-Regular, monospace', defaultTheme: 'system', radius: 'medium' },
  nav: [{ label: 'Docs', href: '/docs/' }],
  actions: [{ label: 'Dashboard', href: '/docs/' }],
  socials: [{ label: 'GitHub', href: 'https://github.com/prosefly/astro-theme-lotus', external: true, icon: 'github' }],
  docs: { basePath: '/docs', sections: [{ slug: 'guide', label: 'Guide', order: 1 }] },
  footer: { copyright: 'Copyright © 2026 Prosefly.', sections: [] },
} satisfies LotusThemeConfig;

export default themeConfig;
```

## `site`

The `site` object defines the project identity used across the docs shell.

- `title` is the primary site name shown in page chrome.
- `description` is the short summary used in the homepage and metadata.
- `logo` points to the brand asset served from `public/`.

## `appearance`

`appearance` controls the shared visual tokens.

- `accent` selects the primary interactive color family.
- `gray` selects the neutral palette used by surfaces and borders.
- `fontSans` and `fontMono` set the runtime CSS variables for typography.
- `defaultTheme` accepts `light`, `dark`, or `system`. The system option renders
  `data-theme="system"` and follows `prefers-color-scheme` in CSS.
- `radius` accepts the supported radius scale and becomes the
  `data-radius` attribute on the rendered page shell.

## `nav`

`nav` is an ordered array of header links. Use it for top-level destinations
such as docs, component references, or external properties.

```ts
nav: [
  { label: 'Docs', href: '/docs/' },
  { label: 'References', href: '/docs/references/design-system/' },
]
```

## `actions`

`actions` renders button-like call-to-action links in the header. Actions
support the same link fields as `nav`, plus optional `variant` and `color`
fields. The default is a neutral soft button.

```ts
actions: [
  { label: 'Dashboard', href: '/docs/', variant: 'soft', color: 'neutral' },
  { label: 'Login', href: '/login/', variant: 'solid', color: 'accent' },
]
```

## `socials`

`socials` renders compact icon links after the actions. Supported icon names
include `github`, `x`, `bluesky`, `discord`, and `external`. You can also use a
full Iconify name such as `simple-icons:github`.

```ts
socials: [
  { label: 'GitHub', href: 'https://github.com/prosefly/astro-theme-lotus', external: true, icon: 'github' },
]
```

## `iconify`

Lotus scans static `<Icon name="...">` usage in Astro and MDX files, combines
those names with `socials.icon`, and preloads Iconify data by prefix. Dynamic
icon names still work, but they fall back to an individual request.

```ts
iconify: {
  apiBase: 'https://api.iconify.design',
  preload: ['lucide:search'],
  scan: true,
}
```

## `docs`

`docs.basePath` controls where Lotus injects its catch-all route. Use `/docs`
for `/docs/`, or another path such as `/guide` if the documentation should live
elsewhere.

`docs.sections` defines the top-level section labels and ordering used by the
subnav. Each docs page then opts into one section through frontmatter.

```ts
docs: {
  basePath: '/docs',
  sections: [
    { slug: 'guide', label: 'Guide', order: 1 },
    { slug: 'components', label: 'Components', order: 2 },
  ],
}
```

The sidebar is content-driven: once a page sets `section: guide`, it will be
grouped under that section automatically and sorted by its `order`.

Sections can also define an explicit sidebar. Use `links` for resource links at
the top of the sidebar, `groups` for titled sections, and nested `items` for
collapsible rows.

```ts
docs: {
  sections: [
    {
      slug: 'guide',
      label: 'Guide',
      sidebar: {
        links: [
          { label: 'GitHub', href: 'https://github.com/prosefly/astro-theme-lotus', external: true, icon: 'github' },
          { label: 'Blog', href: '/docs/references/markdown/', icon: 'lucide:newspaper' },
        ],
        groups: [
          {
            title: 'Getting Started',
            items: [
              { label: 'Overview', href: '/docs/guide/getting-started/' },
              { label: 'Configuration', href: '/docs/guide/configuration/' },
            ],
          },
          {
            title: 'Components',
            items: [
              {
                label: 'Content blocks',
                items: [
                  { label: 'Callout', href: '/docs/components/callout/' },
                  { label: 'Steps', href: '/docs/components/steps/' },
                ],
              },
            ],
          },
        ],
      },
    },
  ],
}
```

## `footer`

Use `footer` to define the copyright line and grouped footer links.

```ts
footer: {
  copyright: 'Copyright © 2026 Prosefly.',
  sections: [
    {
      title: 'Docs',
      links: [{ label: 'Getting Started', href: '/docs/guide/getting-started/' }],
    },
  ],
}
```
