# Navigation

Lotus derives subnav from top-level docs sections and scopes the sidebar to the
active section.

## Top-level sections

The top navigation inside the docs experience comes from
`src/theme.config.ts`:

```ts
docs: {
  sections: [
    { slug: 'guide', label: 'Guide', order: 1 },
    { slug: 'components', label: 'Components', order: 2 },
    { slug: 'references', label: 'References', order: 3 },
  ],
}
```

Each entry becomes a subnav item. Lotus sorts these entries by `order`, then by
label.

## Content-driven sidebar

The sidebar is generated from docs content, not from a second navigation file.
Pages opt into a section with frontmatter and are then grouped automatically.

```md
---
title: Configuration
section: guide
order: 2
---
```

When you visit a guide page, Lotus reads all docs entries where `section: guide`
and builds the sidebar for that section only. The same pattern applies to
`components` and `references`.

## Practical effect

- Add a page in an existing section to have it appear in that section sidebar.
- Change `order` to reorder pages within the active section.
- Update `docs.sections` when you want to rename, reorder, or add a new
  top-level section.

Because subnav and sidebar content share the same docs collection, navigation
stays aligned with the pages that actually exist.
