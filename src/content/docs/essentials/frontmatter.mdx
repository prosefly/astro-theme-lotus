Each docs page starts with frontmatter. Lotus uses it for metadata, section
membership, sorting, and page descriptions.

```md
---
title: Installation
description: Install Lotus and register the integration.
section: guide
order: 2
---
```

## Required fields

`title` is required. It becomes the page heading in the docs layout and the
label used by content-driven sidebar navigation.

## Section

`section` tells Lotus which top-level sidebar owns the page. If it is omitted
for a nested page, Lotus falls back to the first path segment.

Each top-level `sidebars` entry generates a docs subnav section. Lotus matches
frontmatter `section` against the sidebar `slug`, or against the slug generated
from the sidebar label.

## Order

`order` controls autogenerated sidebar sorting. Explicit sidebars in
`theme.config.ts` can override the order by listing items directly.
