Lotus is an installable documentation theme for Astro v7, Tailwind CSS v4, and
MDX. It provides the page shell, docs route, navigation model, theme tokens, and
shared MDX components needed to start a project documentation site without
rebuilding the chrome from scratch.

## What Lotus includes

- A sticky main header and docs subnav.
- A content layout with sidebar, main content, table of contents, and footer.
- A built-in search dialog with a generated JSON index.
- A Markdown source route for every docs page.
- Configurable site metadata, nav links, actions, social icons, footer links,
  color tokens, font tokens, and radius.
- MDX support through the Lotus integration.
- Shared Prosefly components for callouts, accordions, steps, cards, badges,
  tabs, file trees, and Iconify icons.

## How the docs tree works

Write docs pages under the locale directory configured in `locales`. This demo
keeps English docs in `src/content/docs/en/` and Simplified Chinese docs in
`src/content/docs/zh-cn/`. The `sidebars` config decides which sidebar and
subnav section owns each page.

Lotus injects the docs routes from `docsBase`. With the default
`docsBase: '/docs'`, the integration generates:

- `/docs/` and nested docs pages
- `/docs/*.md` Markdown source routes
- `/docs/search.json` search index data

This example site uses `docsBase: '/docs'` and a custom loader base of
`./src/content`. With `homepage: 'auto'`, the top-level
`src/content/index.mdx` renders as the site homepage at `/`, while localized
docs render under `/docs/...`.

```txt
src/content/
  index.mdx
  docs/
    en/
      index.mdx
      overview.mdx
      installation.mdx
      configuration/
        project.mdx
        appearance.mdx
        navbar.mdx
      deployment.mdx
      essentials/
        markdown-syntax.mdx
        images.mdx
        code-blocks.mdx
        frontmatter.mdx
      customization/
        css-styling.mdx
        overriding-components.mdx
      components/
        callout.mdx
        steps.mdx
    zh-cn/
      index.mdx
      overview.mdx
```

## First steps

Start with Installation, then configure site identity, appearance, and the
navbar. Once the shell is in place, replace the example pages with your own
product or project docs.
