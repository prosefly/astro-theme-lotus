import { Steps } from '@prosefly/astro-components';

Install the theme, register the integration, connect the content collection,
and start the Astro development server.

<Steps>

1. **Install the packages.**

   Add the theme and shared components package to an Astro project.

   ```sh
   npm install @prosefly/astro-theme-lotus @prosefly/astro-components
   ```

2. **Add the integration.**

   Configure Astro with the Lotus integration.

   ```ts title="astro.config.ts"
   import { defineConfig } from 'astro/config';
   import lotus from '@prosefly/astro-theme-lotus';
   import themeConfig from './src/theme.config';

   export default defineConfig({
     integrations: [lotus(themeConfig)],
   });
   ```

3. **Create the theme config.**

   Add the project identity and initial sidebar structure.

   ```ts title="src/theme.config.ts"
   import { defineLotusConfig } from '@prosefly/astro-theme-lotus';

   export default defineLotusConfig({
     name: 'My Docs',
     description: 'Documentation for my project.',
     navbar: [{ label: 'Docs', href: '/docs/' }],
     sidebars: [
       {
         label: 'Guides',
         items: ['index'],
       },
     ],
   });
   ```

4. **Register the content collection.**

   Create `src/content.config.ts` and use the loader and schema from Lotus.

   ```ts title="src/content.config.ts"
   import { defineCollection } from 'astro:content';
   import { docsLoader, docsSchema } from '@prosefly/astro-theme-lotus/content';

   const docs = defineCollection({
     loader: docsLoader(),
     schema: docsSchema(),
   });

   export const collections = { docs };
   ```

5. **Start development.**

   Lotus reads from `src/content/docs` by default. The integration also injects
   a Markdown source route for each page and a search index route under the same
   `docsBase`.

   ```sh
   npm run dev
   ```

</Steps>

Use `npm run check` before shipping changes. It validates content collections,
MDX imports, and TypeScript.
