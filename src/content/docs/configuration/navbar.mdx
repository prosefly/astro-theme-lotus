Lotus separates navigation into header links, the docs base path, sidebars, and
footer sections.

## Header nav

`navbar` is an ordered array of top-level destinations. Items without a
`variant` render as text links. Items with a `variant` render as button-like
actions.

```ts
navbar: [
  { label: 'Docs', href: '/docs/' },
  { label: 'Components', href: '/docs/components/callout/' },
  { label: 'Dashboard', href: '/dashboard/', variant: 'soft' },
  {
    label: 'Get started',
    href: '/docs/overview/',
    variant: 'solid',
    color: 'accent',
    trailingIcon: 'lucide:chevron-right',
  },
]
```

## Actions and socials

Button-like actions live in `navbar` alongside regular links. `socials`
renders compact icon links after the navbar items.

```ts
navbar: [
  { label: 'Docs', href: '/docs/' },
  { label: 'Dashboard', href: '/docs/', variant: 'soft', color: 'neutral' },
]

socials: [
  { label: 'GitHub', href: 'https://github.com/prosefly/astro-theme-lotus', external: true, icon: 'github' },
]
```

Icon names can be short aliases such as `github`, `x`, `discord`, and
`external`, or full Iconify names such as `simple-icons:github`.

## Docs base

`docsBase` controls the route where Lotus injects the docs pages.

```ts
docsBase: '/docs'
```

Use a different value such as `/guide`, `/manual`, or `/` when the docs should
live somewhere else.

## Sidebars

`sidebars` uses a Starlight-like structure. Top-level sidebars generate the
docs subnav, while their `items` define the sidebar shown for that section.
Items can be string shortcuts, external links, nested groups, or autogenerated
directories.

```ts
sidebars: [
  {
    slug: 'guide',
    label: 'Guides',
    icon: 'lucide:rocket',
    items: [
      {
        label: 'GitHub',
        link: 'https://github.com/prosefly/astro-theme-lotus',
        icon: 'simple-icons:github',
      },
      {
        label: 'Getting Started',
        items: [
          'overview',
          'installation',
          {
            label: 'Configuration',
            items: [{ autogenerate: { directory: 'configuration' } }],
          },
          'deployment',
        ],
      },
    ],
  },
  {
    label: 'Components',
    icon: 'lucide:blocks',
    items: [{ autogenerate: { directory: 'components' } }],
  },
]
```

`slug` is optional. If omitted, Lotus derives it from the label, so `Components`
becomes `components`. Add `slug` when you need a stable subnav id that differs
from the visible label, such as `label: 'Guides'` with `slug: 'guide'`.

`icon` is also optional. When present, it appears in the docs subnav before the
section label.

String items resolve against `src/content/docs` entry IDs. For example,
`'installation'` resolves to `src/content/docs/installation.mdx`.

```ts
{
  label: 'Essentials',
  items: [{ autogenerate: { directory: 'essentials' } }],
}
```

Autogenerated directories include every entry under that directory and sort
them by frontmatter `order`, then by title.

Nested `items` render as collapsible rows inside their parent group.

```ts
{
  label: 'Getting Started',
  items: [
    {
      label: 'Configuration',
      items: [
        { autogenerate: { directory: 'configuration' } },
      ],
    },
  ],
}
```

New content should prefer `sidebar.order` in frontmatter, but the older
top-level `order` field is still accepted as a shorthand.

## Footer

`footer` controls the lower site chrome. Use `copyright` for the small text
line and `sections` for grouped links.

```ts
footer: {
  copyright: 'Copyright © 2026 Prosefly.',
  sections: [
    {
      title: 'Docs',
      links: [
        { label: 'Overview', href: '/docs/overview/' },
        { label: 'Installation', href: '/docs/installation/' },
      ],
    },
    {
      title: 'Project',
      links: [
        {
          label: 'GitHub',
          href: 'https://github.com/prosefly/astro-theme-lotus',
          external: true,
        },
      ],
    },
  ],
}
```

Footer links use the same `label`, `href`, and optional `external` shape as
other theme links.
