`navbar` controls the main header links. Regular items render as text links.
Items with `variant` render as button-like actions.

```ts
navbar: [
  { label: 'Docs', href: '/docs/' },
  { label: 'Components', href: '/docs/components/icon/' },
  { label: 'References', href: '/docs/references/design-system/' },
  { label: 'Dashboard', href: '/dashboard/', variant: 'soft' },
  {
    label: 'Get started',
    href: '/docs/overview/',
    variant: 'solid',
    color: 'accent',
    trailingIcon: 'lucide:chevron-right',
  },
]
```

## Links

Every navbar item needs `label` and `href`.

```ts
{ label: 'Docs', href: '/docs/' }
```

Add `external: true` when a link should open in a new tab.

## Action Buttons

Add `variant` to render a navbar item as a button.

```ts
{ label: 'Dashboard', href: '/dashboard/', variant: 'soft' }
```

Supported variants are `soft`, `outline`, and `solid`.

`color` can be `neutral` or `accent`. If omitted, navbar actions default to a
neutral soft style.

## Icons

Navbar items can use `icon` and `trailingIcon`.

```ts
{
  label: 'Get started',
  href: '/docs/overview/',
  variant: 'solid',
  color: 'accent',
  trailingIcon: 'lucide:chevron-right',
}
```

Icon values can be full Iconify names or aliases resolved by Lotus, such as
`github`, `x`, `discord`, and `external`.

## Social Links

`socials` renders compact icon links after the navbar.

```ts
socials: [
  {
    label: 'GitHub',
    href: 'https://github.com/prosefly/astro-theme-lotus',
    external: true,
    icon: 'github',
  },
  {
    label: 'X',
    href: 'https://x.com/prosefly',
    external: true,
    icon: 'x',
  },
]
```

Social links require an `icon`.

## Mobile Header

On mobile, Lotus keeps search and the menu trigger in the header. Navbar links,
actions, socials, and theme controls move into the mobile menu.
