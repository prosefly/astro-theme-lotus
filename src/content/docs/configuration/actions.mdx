Header buttons are navbar items with a `variant`. They are useful for
destinations such as login, dashboard, API console, or a product app.

```ts
navbar: [
  { label: 'Docs', href: '/docs/' },
  { label: 'Dashboard', href: '/docs/', variant: 'soft', color: 'neutral' },
  {
    label: 'Login',
    href: '/login/',
    variant: 'solid',
    color: 'accent',
    trailingIcon: 'lucide:chevron-right',
  },
]
```

## Variants

Navbar buttons support `soft`, `outline`, and `solid` variants. Regular navbar
links omit `variant`.

## Social icons

Social links use the shared `Icon` component and Iconify data:

```ts
socials: [
  { label: 'GitHub', href: 'https://github.com/prosefly/astro-theme-lotus', external: true, icon: 'github' },
  { label: 'X', href: 'https://x.com/prosefly', external: true, icon: 'x' },
]
```

Lotus preloads icons from the theme configuration and from static `<Icon
name="...">` usage in Astro and MDX files.

## Page actions

Page actions appear near the article title. The default actions copy the current
page, open the Markdown version, and pass the page URL to ChatGPT or Claude.

```ts
pageActions: [
  { type: 'copy-page' },
  { type: 'view-markdown' },
  { type: 'open-chatgpt' },
  { type: 'open-claude' },
]
```

Set `pageActions` to an empty array to hide the control:

```ts
pageActions: []
```

Each action can override its label, icon, and URL:

```ts
pageActions: [
  { type: 'copy-page', label: 'Copy' },
  {
    label: 'Open issue',
    icon: 'lucide:bug',
    href: 'https://github.com/prosefly/astro-theme-lotus/issues/new?title={encodedTitle}',
    external: true,
  },
]
```

Custom `href` values can use `{title}`, `{url}`, `{markdownUrl}` and their
encoded variants: `{encodedTitle}`, `{encodedUrl}`, `{encodedMarkdownUrl}`.
