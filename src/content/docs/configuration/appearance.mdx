`appearance` controls the shared visual tokens that Lotus applies to layouts
and package components.

```ts
appearance: {
  accent: 'indigo',
  gray: 'neutral',
  defaultMode: 'system',
  radius: 'medium',
}
```

## Theme mode

`defaultMode` accepts `light`, `dark`, or `system`. The system option renders
`data-theme="system"` and follows the visitor's `prefers-color-scheme` setting.

## Color

`gray` selects the Tailwind neutral scale used by text, surfaces, dividers, code
blocks, and other low-emphasis UI. Supported values are `slate`, `zinc`,
`neutral`, and `stone`. Lotus renders this as an attribute such as
`data-gray="neutral"` and maps that selector to aliases like
`--lotus-gray-500: var(--color-neutral-500)`. Semantic tokens such as
`--lotus-text` and `--lotus-surface` are then derived from that scale. Lotus
keeps the supported Tailwind color variables available with `@theme static`, so
projects do not need extra Tailwind configuration for the default palettes.

`accent` selects the color used by active navigation, links, focus rings, and
primary actions. Built-in accents use Tailwind color scales and render as a
document attribute such as `data-accent="indigo"`.

Supported built-in accents are `red`, `orange`, `amber`, `yellow`, `lime`,
`green`, `emerald`, `teal`, `cyan`, `sky`, `blue`, `indigo`, `violet`,
`purple`, `fuchsia`, `pink`, and `rose`.

You can also pass a custom hex color such as `#3b82f6`. Lotus converts it into
light and dark accent variables so the same hue works across color modes.

## Radius

`radius` controls the `data-radius` attribute on the page shell. Lotus maps that
attribute to radius tokens so buttons, cards, sidebar items, and MDX components
share the same shape language.

## Fonts

Lotus does not register fonts by default. The theme uses system sans and mono
stacks through `--lotus-font-sans` and `--lotus-font-mono`.

Projects can load fonts with Astro, a CSS import, or any other provider, then
override the font tokens in their own CSS:

```css
:root {
  --lotus-font-sans: "Inter", var(--lotus-system-sans);
  --lotus-font-mono: "JetBrains Mono", var(--lotus-system-mono);
}
```
