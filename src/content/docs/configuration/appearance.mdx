`appearance` controls the shared visual tokens that Lotus applies to layouts
and package components.

```ts
appearance: {
  accent: 'indigo',
  gray: 'neutral',
  fontSans: 'Inter, ui-sans-serif, system-ui, sans-serif',
  fontMono: 'JetBrains Mono, ui-monospace, SFMono-Regular, monospace',
  defaultTheme: 'system',
  radius: 'medium',
}
```

## Theme mode

`defaultTheme` accepts `light`, `dark`, or `system`. The system option renders
`data-theme="system"` and follows the visitor's `prefers-color-scheme` setting.

## Color

`gray` selects the Tailwind neutral scale used by text, surfaces, dividers, code
blocks, and other low-emphasis UI. Supported values are `slate`, `zinc`,
`neutral`, and `stone`. Lotus renders this as an attribute such as
`data-gray="neutral"` and maps that selector to aliases like
`--lotus-gray-500: var(--color-neutral-500)`. Semantic tokens such as
`--lotus-text` and `--lotus-surface` are then derived from that scale. Lotus
keeps the supported Tailwind color variables available with `@theme static`, so
projects do not need extra Tailwind configuration for the default palettes.

`accent` selects the Tailwind color used by active navigation, links, focus
rings, and primary actions. Built-in accents include `indigo`, `blue`, `violet`,
`purple`, `emerald`, `teal`, `rose`, and `orange`. You can also pass a CSS color
value when a project needs a custom accent.

## Radius

`radius` controls the `data-radius` attribute on the page shell. Lotus maps that
attribute to radius tokens so buttons, cards, sidebar items, and MDX components
share the same shape language.

## Fonts

Use `fontSans` and `fontMono` to set runtime CSS variables. Lotus does not load
remote fonts for you, so add font files or external font styles in the host
project when needed.
