# Icon

`Icon` renders SVG icons from Iconify data. Pass a full Iconify name in the
`prefix:icon` format.

## Import

```mdx
import { Icon } from '@prosefly/astro-components';
```

## Example

<div class="inline-flex items-center gap-2">
  <Icon name="lucide:star" title="Star" />
  <span>Favorite this page</span>
</div>

## Props

| Prop | Type | Default | Notes |
| --- | --- | --- | --- |
| `name` | `string` | required | Iconify name such as `lucide:star`. |
| `title` | `string` | optional | Adds an accessible SVG title. |
| `size` | `string \| number` | `1em` | Sets width and height together. |
| `width` | `string \| number` | `size` | Overrides SVG width. |
| `height` | `string \| number` | `size` | Overrides SVG height. |
| `apiBase` | `string` | `https://api.iconify.design` | Iconify API endpoint. |

## Notes

Icon data is loaded from the Iconify API during server rendering or static
build. Builds therefore need network access unless you provide an internal
Iconify-compatible API endpoint with `apiBase`.

When the Lotus integration is installed, it preloads static Icon usage in
Astro and MDX files and groups requests by Iconify prefix. For example,
`lucide:star` and `lucide:search` can be fetched through one `lucide` request.
Icons that cannot be detected statically still fall back to an individual API
request.

The preload behavior lives in the components package and can also be installed
directly through the `@prosefly/astro-components/icons` integration. Lotus uses
that integration internally and adds icons from the theme configuration.
