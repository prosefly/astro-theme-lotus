import { TabItem, Tabs } from '@prosefly/astro-components';

`Tabs` and `TabItem` group related variants while keeping each panel addressable
as accessible tab content. The authoring syntax follows Starlight: put one or
more `TabItem` components inside `Tabs`.

## Import

```mdx
import { TabItem, Tabs } from '@prosefly/astro-components';
```

## Example

<Tabs syncKey="package-manager">
  <TabItem label="pnpm" icon="simple-icons:pnpm">
    ```sh
    pnpm install
    pnpm dev
    ```
  </TabItem>
  <TabItem label="npm" icon="simple-icons:npm">
    ```sh
    npm install
    npm run dev
    ```
  </TabItem>
  <TabItem label="yarn" icon="simple-icons:yarn">
    ```sh
    yarn install
    yarn dev
    ```
  </TabItem>
</Tabs>

## Without Icons

<Tabs>
  <TabItem label="Preview">
    The preview tab can contain prose, lists, code blocks, and other MDX
    components.
  </TabItem>
  <TabItem label="Code">
    ```astro
    <Tabs>
      <TabItem label="Preview">Content</TabItem>
      <TabItem label="Code">Code sample</TabItem>
    </Tabs>
    ```
  </TabItem>
</Tabs>

## Props

### `Tabs`

| Prop | Type | Default | Notes |
| --- | --- | --- | --- |
| `syncKey` | `string` | optional | Persists and syncs the active label across tab groups with the same key. |

### `TabItem`

| Prop | Type | Default | Notes |
| --- | --- | --- | --- |
| `label` | `string` | required | Visible tab trigger label and the value used by `syncKey`. |
| `icon` | `string` | optional | Iconify icon name. When present, the active tab icon appears in the top-left of the tab group. |

## Notes

Use the same labels across groups that share a `syncKey`. For example, package
manager tabs should use the same `pnpm`, `npm`, and `yarn` labels everywhere so
one selection can update all matching groups.
