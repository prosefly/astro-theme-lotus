Use `Steps` for procedural content that benefits from a consistent rhythm, a
visible guide line, and scannable numbering.

## Import

```mdx
import { Steps } from '@prosefly/astro-components';
```

## Example

<Steps>

1. Create the content file.

   Add a new MDX page under `src/content/docs/` and include frontmatter for
   `title`, `description`, and sidebar ordering.

2. Import the components you need.

   Import from the shared MDX entrypoint so examples stay consistent across the
   docs site.

   ```mdx
   import { Steps } from '@prosefly/astro-components';
   ```

3. Render the pattern inline.

   Write the procedural content directly as a Markdown ordered list. Each item
   can include paragraphs, nested lists, and code blocks.

</Steps>

## Syntax

`Steps` expects exactly one ordered Markdown list as its content.

```mdx
<Steps>

1. First step.

   Supporting content for the first step.

2. Second step.

   - Nested lists are supported.
   - Code blocks are supported.

</Steps>
```

You can use the standard ordered-list `start` attribute when numbering needs to
continue from a previous sequence.

```mdx
<Steps>

4. Continue with the fourth step.
5. Finish the sequence.

</Steps>
```
